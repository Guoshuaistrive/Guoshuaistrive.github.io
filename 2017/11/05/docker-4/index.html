初学docker必看总结

发表于 2017-11-05 | 分类于 summary | 阅读次数:

前言

这篇文章主要介绍了初次使用Docker的体验总结，对初学Docker具有一定的参考价值，有需要的可以了解一下。以下内容大部分是主观片面的，可根据自己情况执行。

环境

我的基础运行环境是运行在Win7虚拟机中的Ubuntu版本，Ubuntu安装Docker简洁明快：

sudo apt-get install docker

获取容器镜像

Docker提供镜像搜索命令（search），方便你快速找到自己想要的镜像。搜索命令如下： 

guoshuai@guoshuai:~$ sudo docker search ubuntu
NAME                STARS     OFFICIAL   AUTOMATED         DESCRIPTION              

ubuntu               6768      [OK]                       Ubuntu is a Debian-based 

Linux operating s...    141                  [OK]      
dorowu/ubuntu-desktop-lxde-vnc                         Ubuntu with openssh-

server and NoVNC       115                  [OK]      
rastasheep/ubuntu-sshd                                 Dockerized SSH service,

可以看到，搜索到好多镜像。我选择第一个，标记着“官方的”，应该没有装任何其他的东西。接下来就是下载（pull）镜像了。命令如下（时间略长）：

guoshuai@guoshuai:~$ sudo docker pull ubuntu
Using default tag: latest
latest: Pulling from library/ubuntu

ae79f2514705: Pull complete 
c59d01a7e4ca: Pull complete 
41ba73a9054d: Pull complete 
f1bbfd495cc1: Pull complete 
0c346f7223e2: Pull complete 
Digest: sha256:6eb24585b1b2e7402600450d289ea0fd195cfb76893032bbbb3943e041ec8a65
Status: Downloaded newer image for ubuntu:latest

可以看到，默认下载的是最新的（latest）。当然，你可以下载其他非最新的版本，具体操作请查询pull命令的相关参数。

创建容器

接下来就是从刚下载的镜像创建一个容器了。创建容器（run）命令就比较复杂了，参数较多：

guoshuai@guoshuai:~$ sudo docker run -itd --name strive ubuntu /bin/bash
[sudo] guoshuai 的密码： 
136c6463541d4e1c3dbd1924a2ee886f2bcc15f8d167d71c5091cf87221bd256

如上，我来一个一个解释：

  • -itd：这是 -i -t -d 的简写。-i 表示保持打开标准输入流（stdin），无论是否连接
    到此容器。-t 为容器分配一个虚拟的tty。-d 表示后台模式运行容器，即容器启动后将断
    开与当前终端的连接。
  • –name：这是为容器起一个名字，之后与容器的交互需要用到。
  • ubuntu：这个是本地镜像名称（ubuntu），必须是已经下载（pull）到本地的镜像。当
    然，若是本地没有，Docker会执行search并询问你是不是要下载。
  • /bin/bash：这个是本次启动的镜像要执行的任务。

查看容器

使用docker ps -a 命令可以查看当前存在的所有容器：

guoshuai@guoshuai:~$ sudo docker ps -a
CONTAINER ID        IMAGE       COMMAND         CREATED             STATUS             NAMES
136c6463541d        ubuntu     "/bin/bash"      About a minute ago  Up About a minute  strive

如上，我只运行了一个容器。命令显示了容器的ID、使用的镜像、运行的命令、创建时间、运行时间、名称。

连接容器

连接容器有多种办法。在宿主机中使用Docker的attach命令，即可连接到容器：

guoshuai@guoshuai:~$ sudo docker attach strive
root@136c6463541d:/# 
root@136c6463541d:/# ls
bin  boot  dev  etc  home  lib  lib64  media  mnt  opt  proc  root  run  sbin  

srv  sys  tmp  usr  var
root@136c6463541d:/# ps -aux
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  18240  3452 ?        Ss   13:55   0:00 /bin/bash
root        10  0.0  0.1  34424  2864 ?        R+   14:01   0:00 ps -aux

如上，就已经连接到容器了。可以看到命令行信息的变化。我使用了ps命令查看了容器中的进程。可以看到一个ID为1的进程，这个进程就是run命令最后的那个参数。Run命令所指定的任务进程Id一定是1 。我们使用attach命令连接的也是这个命令的标准输入输出流（错误流我不知道连接了没）。由于run指定的命令是bash，所以attach也是连接到了bash，这样就能随意操作容器了。

关闭连接很容易想到，就是退出bash：exit。但是这样退出会导致一个问题，那时就是：容器因为1号进程的退出（就是attach命令连接的bash），会认为任务已完成，容器就停止了。

root@136c6463541d:/# exit
exit
guoshuai@guoshuai:~$ sudo docker ps -a
CONTAINER ID    IMAGE     COMMAND       CREATED        STATUS                       NAMES
136c6463541d    ubuntu    "/bin/bash"   9 minutes ago  Exited (0) 2 minutes ago     strive

如上，容器已经停止了（Exited）。

所以，我建议使用exec命令对容器进行操作。我们重启容器：

guoshuai@guoshuai:~$ sudo docker start strive
strive
guoshuai@guoshuai:~$ sudo docker ps -a
CONTAINER ID    IMAGE    COMMAND        CREATED          STATUS         NAMES
136c6463541d    ubuntu   "/bin/bash"    10 minutes ago   Up 6 seconds   strive

可以看到，start 加容器名称就能启动。使用exec命令，可以让容器直接执行命令（docker exec 容器名称 命令）：docker exec strive ps -aux

guoshuai@guoshuai:~$ sudo docker exec strive ps -aux
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  18236  3300 ?        Ss+  14:05   0:00 /bin/bash
root         9  0.0  0.1  34424  2860 ?        Rs   14:08   0:00 ps -aux
guoshuai@guoshuai:~$ sudo docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED            STATUS       NAMES
136c6463541d        ubuntu              "/bin/bash"         15 minutes ago     Up 5 minutes   strive

上图可见，我退出了容器的bash后，容器依然在运行。我在宿主机中使用exec执行ps命令后，strice这个容器中1号进程仍然在运行。

停止容器

上面提到了启动容器（start）命令，停止容器当然是停止命令（stop）了。

guoshuai@guoshuai:~$ sudo docker stop strive
strive
guoshuai@guoshuai:~$ sudo docker ps -a
CONTAINER ID    IMAGE    COMMAND        CREATED          STATUS                       NAMES
136c6463541d    ubuntu   "/bin/bash"    17 minutes ago   Exited (0) 6 seconds ago     strive

stop命令执行效果如上。
