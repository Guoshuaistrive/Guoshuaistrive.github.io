肖德时文章笔记（一）

发表于 2017-11-25 | 分类于 learn | 阅读次数:

接触docker大概半个月的时间，以为了解了个大概，但是今天看了肖德时老师的文章感觉需要深入学习的太多，文章很多部分还需再学习，所以先简要做了一些记录。

Docker是Docker Inc公司开源的一项基于Ubuntu LXC技术之上构建的应用打包运行时引擎，源代码托管在GitHub上，完全基于go语言开发并遵守Apache2.0协议开源。

Docker 的技术细节

隔离性：namespace 将 container 的进程、网络、消息、文件系统、UTS（“UNIX Time-sharing System”）和用户空间隔离开。

  • pid namespace

不同用户的进程就是通过 pid namespace 隔离开的，且不同 namespace 中可以有相同 pid。

  • net namespace

有了 pid namespace，每个 namespace 中的 pid 能够相互隔离，但是网络端口还是共享 host 的端口。

  • ipc namespace

container 中进程交互还是采用 linux 常见的进程间交互方法（interprocess communication - IPC），包括常见的信号量、消息队列和共享内存。

  • mnt namespace

类似 chroot，将一个进程放到一个特定的目录执行

  • uts namespace

UTS（“UNIX Time-sharing System”）namespace 允许每个 container 拥有独立的 hostname 和 domain name，使其在网络上可以被视作一个独立的节点而非 Host 上的一个进程。

  • user namespace

每个 container 可以有不同的 user 和 group id，也就是说可以在 container 内部用 container 内部的用户执行程序而非 Host 上的用户。

可配额/可度量 Control Groups (cgroups)

cgroups 实现了对资源的配额和度量。

便携性

AUFS（AnotherUnionFS）是一种 Union FS，简单来说就是支持将不同目录挂载到同一个虚拟文件系统下（unite several directories into a single virtual filesystem）的文件系统。

安全性

基于 Docker 的 DevOps 实践

使用 Docker 提供的虚拟化方式，给开发团队建立一套可以复用的开发环境，让开发环境可以通过 Image 的形式分享给项目的所有开发成员，以简化开发环境的搭建。

组建持续发布的敏捷团队

在熟悉了 Docker 命令行之后，团队需要解决几个关键问题具体如下：

  • Base Image 的选择，比如phusion-baseimage

  • 管理Docker应用配置工具的选择，比如Ansible

  • Host主机系统的选择，比如CoreOS、Ubuntu

持续集成的构建系统

需要有一个支持 Docker 的构建系统，这里推荐 Jenkins。它的主要特点是项目开源、方便定制、使用简单。Jenkins 可以方便的安装各种第三方插件，从而方便快捷的集成第三方的应用。

最佳的发布环境

基于 Docker 的私有云，更进一步我们可能期望的是提供 API 接口的 PaaS 云服务。

容器云平台的构建实践

从容器化角度总结起来就是两块，第一是功能需求，管理容器运行引擎、容器编排、容器网络、容器存储、监控报警日志。第二是非功能需求，可用性，兼容性，安全和易用性，负载优化等。容器云平台建设的目标是使企业业务应用被更好的运营管理起来。
