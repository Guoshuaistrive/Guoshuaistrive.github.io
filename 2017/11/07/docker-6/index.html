Dockerfile构建镜像实例

发表于 2017-11-07 | 分类于 exercise | 阅读次数:

Dockerfile

用dockerfile 创建镜像，其内容如下：

[root@i-y61tbtqs ~]# ls
Dockerfile  gitgs  gs
[root@i-y61tbtqs ~]# vi Dockerfile

FROM ubuntu
RUN apt-get update && apt-get install -y vim

用docker build命令构建镜像并分析每个细节：

[root@i-y61tbtqs ~]# docker build -t ubuntu-gs-vi-dockerfile .
Sending build context to Docker daemon 252.4 kB
Step 1 : FROM ubuntu
 ---> dd6f76d9cc90
Step 2 : RUN apt-get update && apt-get install -y vim
 ---> Running in bb053d594b57

......

Processing triggers for libc-bin (2.23-0ubuntu9) ...
 ---> d501007edccf
Removing intermediate container ca9d59d1e6d7
Successfully built d501007edccf
  • 当前目录为/root
  • Dockerfile准备就绪
  • 运行docker build命令，-t将新镜像命名为ubuntu-gs-vi-dockerfile，命令末尾的 . 指明build context为当前目录。
  • Step 1：执行FROM，将Ubuntu作为base镜像。镜像ID为dd6f76d9cc90
  • Step 2：执行RUN，安装vim。（启动ID为bb053d594b57的临时容器，在容器中通过apt-get安装vim）
  • 安装成功后，将容器保存为镜像，其ID为d501007edccf（这一步底层使用的类似docker commit的命令）
  • 删除临时容器bb053d594b57
  • 构建镜像成功

查看镜像分层结构

ubuntu-gs-vi-dockerfile通过在base镜像的顶部添加一个新的镜像层而得到的，这个新的镜像层由RUN apt-get update && apt-get install -y vim生成。这一点可以通过docker history命令验证。

  • 注：missing表示无法获取IMAGE ID，通常从Docker Hub下载的镜像会有这个问题。

镜像的缓存特性

  • Docker会缓存已有镜像的镜像层，构建新镜像时，如果某镜像层已经存在，就直接使用，无须重新创建。

  • 如果希望在构建镜像时不使用缓存，可以在docker build命令中加上–no-cache参数 。

  • Dockerfile中每一个指令都会创建一个镜像层，上层依赖下层。无论什么时候，只要某一层发生变化，其上面所有层的缓存都会失效。
