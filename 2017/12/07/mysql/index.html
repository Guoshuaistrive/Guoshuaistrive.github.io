另一个使用到mysql的容器连接firstsql

[root@guoshuai01 ~]# docker run --name firstapp \
                   --link firstsql:mysql -d httpd:2.4
225bc340c6839192836040a1c68b2362adee8ab2daafdf98074a2d06938c7e34
  • –link A:B 其中A是我们要连接的容器名，B是我们将要运行的这个容器名（A容器ID会添加到B容器的文件目录下面，B负责设置一些与A连接的环境变量）

我们可以进入firstapp里面验证一下是否link成功：

Docker容器里运行应用的数据存储问题

在host上创建一个目录

docker run --name firstsql -v /mysql/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=666666 -d mysql:5.5

  • -v /mysql/data:/var/lib/mysql命令指挂载host上/mysql/data目录到容器目录/var/lib/mysql里

mysql命令行客户端连接firstsql

docker-compose

compose是一个在docker里面定义和运行复杂应用的的工具。我们可以在一个文件里面定义多个容器应用，然后用一条docker-compose命令就可以让应用跑起来。

[root@guoshuai01 opt]# cd mysql/
[root@guoshuai01 mysql]# ls
[root@guoshuai01 mysql]# touch mysql.yml
[root@guoshuai01 mysql]# ls
mysql.yml
[root@guoshuai01 mysql]# vi mysql.yml
  • mysql.yml文件内容如下：
# Use root/example as user/password credentials
version: '3.1'

services:

  db:
    image: mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: example

  adminer:
    image: adminer
    restart: always
    ports:
      - 8080:8080

再执行docker-compose

[root@guoshuai01 mysql]# docker-compose -f mysql.yml up

加载完毕之后，登录http://localhost：8080查看并注册（user/password ：root/example），登录数据库即可操作。
