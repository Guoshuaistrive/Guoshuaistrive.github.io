Centos7.x初始化之后的配置

发表于 2018-01-21 | 分类于 learn | 阅读次数:

查看系统相关信息

查看系统版本

cat /etc/redhat-release

查看系统位数

getconf LONG_BIT

查看内核版本

uname -a

升级内核版本

yum update

yum upgrade

重新启动 init 6 发现版本会有变化（看日期）

selinux设置

检查selinux是否关闭sestatus

如果是enabled状态要执行关闭

sed -i 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config
setenforce 0

再次检查sestatus

hostname

vi /etc/hostname
vi /etc/hosts
hostname -F /etc/hostname

重新登录，查看主机名以及FQDN

hostname
hostname -f

vim

安装配置

yum install vim

修改/etc/profile，添加alias vi='vim'至文件

让修改立即生效

source /etc/profile

vim常用快捷方式

gg  可以将光标移动到文章最前
shift+g 或者 G 可以将光标移动到最后
^   移动到光标所在行的的 行首
$   移动到光标所在行的的 行尾

x  删除光标所在位置的后面一个字符
#x 删除光标所在位置的后面#个字符
X  删除光标所在位置的前面一个字符
#X 删除光标所在位置的前面#个字符 

dd   删除光标所在行   
#dd  从光标所在行开始删除#行

dG 删除当前光标所在行之后的全部内容（ggdG删除全部内容）

yy  复制光标所在行
#yy 复制光标所在行往下#行
p  粘贴到光标所在位置

r 替换光标所在处的字符
冒号模式替换文中的字符 
  :3,13s/old/new/g(3-13行替换)
  :%s/old/new/g(全文替换)

u 回到上一个操作

查找字符 / 或者？  按n往后或者往前寻找关键字

ssh免密登录

创建公钥和私钥 ssh-keygen -t rsa

复制公钥到远程主机 ssh-copy-id -i /root/.ssh/id_rsa.pub root@192.168.0.10

输入远程主机的密码，即可把本地公钥追加到远程主机/root/.ssh/authorized_keys文件中

改变权限

chmod 600 id_rsa
chmod 600 authorized_keys

尝试远程登录，发现不需要密码即可进入

chrony同步时间

安装配置

yum install chrony
systemctl start chronyd
ststemctl enable chronyd

默认同步时间的服务器是Centos，若有需要我们可以改为国内时间服务器

vi /etc/chrony.conf
   server 0.cn.pool.ntp.org iburst
   server 1.cn.pool.ntp.org iburst
   server 2.cn.pool.ntp.org iburst
   server 3.cn.pool.ntp.org iburst

重启chronyd服务

systemctl restart chronyd
chronyc sources

强制同步时间

systemctl stop chronyd
chronyd -q 'pool pool.ntp.org iburst'
systemctl start chronyd

Date&Time

timedatectl status

手动修改时间

先停止同步时间timedatectl set-ntp 0

设置时间 timedatectl set-time 17:00:00

启动ntp服务timedatectl set-ntp 1

修改日期 timedatectl set-time 'YYYY-MM-DD HH:MM:SS'

查看硬件时间

hwclock

同步系统时间到硬件hwclock -w

关闭IPv6

关闭前先查看下ip a

执行关闭操作

vi /etc/sysctl.conf
   net.ipv6.conf.all.disable_ipv6 = 1

立即生效 sysctl -p

对比一下ip a

lrzsz工具的使用

yum install lrzsz
rz 上传文件至服务器
sz 下载文件至本地

可以在xshell文件-属性里面设置文件传输的路径以及重复文件处理

Systemd

列出系统所有的服务
systemctl list-units --type service

只列出启动的服务
systemctl list-unit-files --type=service | grep enabled

针对某一个服务的操作

systemctl status name
systemctl stop name
systemctl disable name
systemctl start name
systemctl restart name
